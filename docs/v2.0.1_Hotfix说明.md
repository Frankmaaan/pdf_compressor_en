## 🔍Root Cause Analysis

### Source of the problem
In the `_execute_scheme()` function of `compressor/strategy.py`, `pipeline.reconstruct_pdf()` was called with incorrect parameter names.

### Code comparison

#### ❌ Error Codes (v2.0.0)
```python
# strategy.py line 213
success = pipeline.reconstruct_pdf(
    images=precomputed_data['image_files'], # ❌ Wrong parameter name
    hocr=precomputed_data['hocr_file'], # ❌ Wrong parameter name
    temp_dir=temp_dir,
    params=params,
    output_pdf_path=output_pdf_path
)
```

#### ✅ Correct code (v2.0.1)
```python
# strategy.py line 213
success = pipeline.reconstruct_pdf(
    image_files=precomputed_data['image_files'], # ✅ Correct parameter name
    hocr_file=precomputed_data['hocr_file'], # ✅ Correct parameter name
    temp_dir=temp_dir,
    params=params,
    output_pdf_path=output_pdf_path
)
```

#### 📋 Function signature
```python
# pipeline.py line 114
def reconstruct_pdf(image_files, hocr_file, temp_dir, params, output_pdf_path):
    """Use recode_pdf to reconstruct the PDF."""
```

### Why does this error occur?

1. **Unit testing limitations**:
- Unit testing uses mock objects
- Mock objects do not validate parameter names
- Only the logic flow is verified, the real function is not executed

2. **Integration test missing**:
- No end-to-end integration testing was conducted before v2.0.0 was released
- Not running the complete process in a real environment
- Not tested with real PDF files

3. **Code Review Missing**:
- The parameter name was modified during reconstruction but the calling location was not updated synchronously.
- No parameter inconsistencies were found during code review

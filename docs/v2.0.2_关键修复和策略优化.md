# v2.0.2 Important fixes and strategy optimization

## release date
2025-10-19

## Problem discovery
When testing a 156-page 136MB PDF file in the WSL/Ubuntu 24.04 environment, the following problems were found:

### 1. **KeyError: 'size_mb'** (Critical Bug)
- **Phenomena**: Crash when starting split strategy
- **Cause**: The `all_results` dictionary in `strategy.py` uses the `'size'` field, but `splitter.py` expects the `'size_mb'` field
- **Impact**: All scenes that need to be split after compression failure will collapse.

### 2. **Strategy Logic Flaw** (Logic Flaw)
- **Problem 1**: When the S6 result > 8MB, we continue to try S2-S5, which wastes a lot of time (the actual measurement of compressing a 136MB file takes more than 1 hour)
- **Question 2**: There is no smart target switching mechanism. When the original target of 2MB cannot be achieved but can be split (2MB < size <= 8MB), it should be switched to the 8MB target.
- **Issue 3**: The split strategy does not perform pre-checking and attempts to split even if there is no result < 8MB

### 3. **S6 is not radical enough** (Performance Issue)
- Actual measurement found that S6 (DPI=110, BG-Downsample=6) compressed 136MB → ~15MB, which is still much larger than 8MB
- More radical solutions are needed to explore the compression limit

### ⚡ Strategy Optimization

#### 2. Fails directly when S7 > 8MB
**File**: `compressor/strategy.py` (Lines 128-131)

**New logic**:
```python
# Key check: If the S7 result is larger than 8MB, it means that it cannot be split and it will fail directly.
if s7_size_mb > 8.0:
    logging.error(f"❌ The result of the most aggressive scenario S7 ({s7_size_mb:.2f}MB) is still larger than the 8MB split threshold. Even if it is split, it cannot meet the 2MB target and the task fails.")
    return None, all_results
```

**Effect**: Avoid meaningless S2-S6 attempts and save a lot of time

#### 3. Smart target switching: 2MB → 8MB
**File**: `compressor/strategy.py` (Lines 133-137)

**New logic**:
```python
# If the S7 result is between 2MB and 8MB, the switching target is 8MB (prepared for splitting)
if target_size_mb < 8.0 and s7_size_mb > target_size_mb:
    logging.warning(f"⚠️ S7 result ({s7_size_mb:.2f}MB) exceeds original target ({target_size_mb:.2f}MB) but is less than 8MB.")
    logging.info("🔄 Strategy adjustment: Switch the target to 8MB and find the solution closest to 8MB for subsequent splitting.")
    target_size_mb = 8.0
```

**Effect**: When direct compression to 2MB fails but can be split, automatically find the solution closest to 8MB for splitting

#### 4. Split precondition checks
**File**: `compressor/splitter.py` (Lines 23-32)

**New logic**:
```python
# Pre-check: Make sure there is a compression result smaller than 8MB
all_results = compression_results.get('all_results', {})
if not all_results:
    logging.error("❌ Split failed: No compression results are available for splitting.")
    return False

under_8mb = [res for res in all_results.values() if res.get('size_mb', float('inf')) <= 8.0]
if not under_8mb:
    logging.error("❌ Splitting failed: All compression results are larger than 8MB, and valid splitting cannot be performed.")
    return False

logging.info(f"✓ Found {len(under_8mb)} compressed results smaller than 8MB and can be split.")
```

**Effect**: Avoid entering the split process that is doomed to fail
